Free Hosting with GitHub Pages: Complete Setup Guide

Learn how to host your static website for free using GitHub Pages, configure custom domains with is-pro.dev, and set up automatic deployments from your repository.

## Introduction As a developer on the is-pro.dev platform, you're likely looking for a reliable and cost-effective way to host your static sites. GitHub Pages offers a free hosting solution that integrates seamlessly with your GitHub repositories. In this guide, we'll walk you through the process of setting up GitHub Pages for your project, using a custom domain, and deploying your site to myproject.is-pro.dev. ## Prerequisites Before you begin, ensure you have the following: - A GitHub account - A repository for your project (e.g., myproject) - A basic understanding of Git version control - Your project set up on the is-pro.dev platform with a subdomain (e.g., myproject.is-pro.dev) ## Setting Up GitHub Pages To set up GitHub Pages, follow these steps: 1. Navigate to your repository on GitHub (e.g., github.com/yourusername/myproject). 2. Click on the "Settings" tab. 3. Scroll down to the "GitHub Pages" section. 4. Under "Source", select the branch you want to use for your site (e.g., main). 5. Click "Save" to save your changes. GitHub Pages will now be enabled for your repository, and your site will be accessible at https://yourusername.github.io/myproject. ## Configuring a Custom Domain To use a custom domain (e.g., myproject.is-pro.dev), you'll need to configure DNS settings: 1. Create a new file in your repository's root directory named "CNAME" (without any extension). 2. Add the following line to the CNAME file: myproject.is-pro.dev 3. Commit the changes to your repository. 4. Go to your DNS provider and set up a CNAME record pointing to your GitHub Pages site (e.g., yourusername.github.io). For is-pro.dev subdomains, you can configure the CNAME record as follows: | Type | Name | Value | | --- | --- | --- | | CNAME | myproject | yourusername.github.io | ## Deploying Your Site To deploy your site to GitHub Pages, you'll need to push your changes to the configured branch (e.g., main): 1. Make sure you're in the root directory of your project. 2. Run the command `git add .` to stage all changes. 3. Run the command `git commit -m "Deploy to GitHub Pages"` to commit the changes. 4. Run the command `git push origin main` to push the changes to your repository. Your site should now be live at myproject.is-pro.dev. ## Common Pitfalls When setting up GitHub Pages, be aware of the following common pitfalls: - Forgetting to configure the CNAME file or DNS settings, resulting in your site not being accessible at the custom domain. - Using the wrong branch or repository, leading to deployment issues. - Not committing changes or pushing them to the wrong branch, causing your site to not update correctly. ## Best Practices To ensure a smooth deployment process, follow these best practices: - Use a consistent branch naming convention (e.g., main for production). - Regularly update your repository and push changes to the configured branch. - Test your site after each deployment to ensure everything is working as expected. - Consider using a CI/CD pipeline to automate your deployment process. ## Conclusion In this guide, we've walked you through the process of setting up GitHub Pages for your project on the is-pro.dev platform. By following these steps and best practices, you can ensure a reliable and cost-effective hosting solution for your static site. If you encounter any issues or have further questions, refer to the FAQ section below. ## FAQ Q: Can I use GitHub Pages for dynamic sites? A: No, GitHub Pages is designed for static sites only. For dynamic sites, consider using a different hosting solution. Q: How do I handle SSL certificates for my custom domain? A: GitHub Pages automatically provides SSL certificates for custom domains. Ensure your DNS settings are configured correctly to enable SSL. Q: Can I use a custom domain with a subfolder on GitHub Pages? A: Yes, you can use a custom domain with a subfolder on GitHub Pages. Configure your CNAME file and DNS settings accordingly. Q: How long does it take for my site to be deployed on GitHub Pages? A: Deployment times on GitHub Pages are typically instantaneous. However, DNS changes may take some time to propagate, depending on your DNS provider.

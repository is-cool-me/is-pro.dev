Deploy to a VPS Using SSH and a Custom Subdomain

Configure a subdomain to point to a VPS server, set up Nginx as a reverse proxy, manage SSL certificates with Certbot, and automate deployments with Git hooks.

## Introduction to VPS Deployment Deploying a project to a Virtual Private Server (VPS) using SSH and a custom subdomain offers a high degree of control and flexibility, making it a popular choice among developers. For projects hosted on is-cool-me, such as `myproject.is-pro.dev`, setting up a VPS allows for more advanced configurations and direct management of server resources. This guide will walk through the process of deploying a project to a VPS, configuring Nginx as a reverse proxy, obtaining an SSL certificate with Certbot, and ensuring a secure and efficient deployment. ## Prerequisites for Deployment Before starting the deployment process, ensure you have the following: - A VPS from a provider like DigitalOcean or AWS. - A custom subdomain set up through is-cool-me, such as `myproject.is-pro.dev`. - SSH access to your VPS. - Basic knowledge of Linux command line and Nginx configuration. - A project ready for deployment, with considerations for production environments. ## Step-by-Step Deployment Instructions ### Setting Up the VPS 1. **Connect to your VPS** via SSH: `ssh root@your_vps_ip`. 2. **Update your VPS**: Run `apt update && apt upgrade -y` to ensure your system is up-to-date. 3. **Install Nginx**: Use `apt install nginx -y` to install Nginx. 4. **Configure the Firewall**: Allow HTTP and HTTPS traffic using `ufw allow http` and `ufw allow https`, then enable the firewall with `ufw enable`. ### Configuring Nginx as a Reverse Proxy 1. **Create a new Nginx configuration file** for your project: `nano /etc/nginx/sites-available/myproject`. 2. **Add the following configuration** to the file, adjusting paths and server names as necessary: ```nginx server { listen 80; server_name myproject.is-pro.dev; location / { proxy_pass http://localhost:3000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_cache_bypass $http_upgrade; } } ``` 3. **Create a symbolic link** to enable the configuration: `ln -s /etc/nginx/sites-available/myproject /etc/nginx/sites-enabled/`. 4. **Test and reload Nginx**: Run `nginx -t` to test the configuration, and `systemctl reload nginx` to apply the changes. ### Obtaining an SSL Certificate with Certbot 1. **Install Certbot**: Run `apt install certbot python3-certbot-nginx -y`. 2. **Obtain the SSL certificate**: Use `certbot --nginx -d myproject.is-pro.dev` to get a certificate for your subdomain. 3. **Follow the Certbot prompts** to complete the setup, which may include setting up a redirect from HTTP to HTTPS. ## Common Pitfalls and Solutions - **Nginx Configuration Errors**: Always test your Nginx configuration before reloading. Common issues include syntax errors or incorrect file paths. - **SSL Certificate Issues**: Ensure your domain is properly configured and reachable. Certbot may fail if it cannot verify domain ownership. - **Firewall Configuration**: Double-check that your firewall allows the necessary traffic. Incorrect settings can block access to your site. ## Best Practices for Deployment - **Use Environment Variables**: For sensitive information like database passwords, use environment variables instead of hardcoding them into your scripts. - **Implement Logging and Monitoring**: Tools like Prometheus and Grafana can provide insights into your application's performance and health. - **Regularly Update Dependencies**: Keep your project's dependencies up-to-date to prevent security vulnerabilities. ## Performance Considerations - **Optimize Database Queries**: Efficient database queries can significantly improve your application's performance. - **Use Caching**: Implement caching mechanisms, like Redis, to reduce the load on your server. - **Monitor Resource Usage**: Regularly check CPU, memory, and disk usage to identify potential bottlenecks. ## Troubleshooting Deployment Issues - **Check Nginx Logs**: Look for errors in `/var/log/nginx/error.log` to diagnose issues with your reverse proxy setup. - **Verify Firewall Rules**: Ensure that your firewall is not blocking necessary traffic. - **Test SSL Certificate**: Use tools like SSL Labs' SSL Test to verify the integrity of your SSL certificate. ## Conclusion and Next Steps Successfully deploying your project to a VPS using SSH and a custom subdomain like `myproject.is-pro.dev` involves careful configuration of Nginx, obtaining an SSL certificate with Certbot, and considering performance and security best practices. After completing these steps, your project should be securely and efficiently deployed. For further learning, explore advanced Nginx configurations, security hardening, and performance optimization techniques. ## FAQ 1. **Q: How do I renew my SSL certificate?** A: Certbot automatically renews certificates. You can also manually renew using `certbot renew`. 2. **Q: What if my project uses a non-standard port?** A: Adjust the `proxy_pass` directive in your Nginx configuration to point to the correct port. 3. **Q: Can I use this setup for multiple projects?** A: Yes, create separate Nginx configurations for each project, ensuring unique server names and proxy pass directives. 4. **Q: How do I handle errors and exceptions in my application?** A: Implement logging and error handling mechanisms within your application, and consider using a logging service for centralized monitoring. 5. **Q: Are there any limits to the number of custom subdomains I can set up?** A: Check with is-cool-me for specific limitations on custom subdomains, as policies may vary. ## Further Reading - **Nginx Documentation**: For in-depth guides on Nginx configuration and optimization. - **Certbot Documentation**: Detailed instructions on obtaining and managing SSL certificates with Certbot. - **DigitalOcean Tutorials**: A wide range of tutorials on VPS setup, security, and deployment strategies. - **is-cool-me Documentation**: Specific guides and FAQs related to setting up and managing projects on is-cool-me subdomains.

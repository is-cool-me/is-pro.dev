Deploy to a VPS Using SSH and a Custom Subdomain

Configure a subdomain to point to a VPS server, set up Nginx as a reverse proxy, manage SSL certificates with Certbot, and automate deployments with Git hooks.

## Introduction Deploying a project to a Virtual Private Server (VPS) using SSH and a custom subdomain is a crucial step in making your application accessible to the public. As a developer on the is-pro.dev platform, you can leverage the flexibility of a VPS to host your projects on custom subdomains like myproject.is-pro.dev. This guide will walk you through the process, covering the setup of SSH, configuration of Nginx, implementation of a reverse proxy, and the installation of SSL certificates using Certbot. ## Prerequisites Before you begin, ensure you have the following: - A VPS with a supported operating system (e.g., Ubuntu). - A custom subdomain set up on is-pro.dev (e.g., myproject.is-pro.dev). - Basic knowledge of SSH, Linux command line, and Nginx. - Certbot installed on your VPS for SSL certificate management. - Nginx installed and configured as your web server. ## Step-by-Step Instructions ### Setting Up SSH First, connect to your VPS using SSH. If you're using a Mac or Linux, you can do this directly from the terminal. For Windows, consider using PuTTY or the Windows Subsystem for Linux. ```bash ssh user@your-vps-ip ``` Replace `user` with your actual username and `your-vps-ip` with the IP address of your VPS. ### Configuring Nginx Create a new configuration file for your project in the Nginx sites-available directory. For example, for `myproject.is-pro.dev`, you might create a file named `myproject`: ```bash sudo nano /etc/nginx/sites-available/myproject ``` Add the following configuration to the file, adjusting the server name and root directory as necessary: ```nginx server { listen 80; server_name myproject.is-pro.dev; location / { proxy_pass http://localhost:3000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_cache_bypass $http_upgrade; } } ``` This configuration sets up a reverse proxy, directing traffic from `myproject.is-pro.dev` to your application running on `http://localhost:3000`. ### Enabling the Nginx Configuration Create a symbolic link to the `sites-enabled` directory to enable the configuration: ```bash sudo ln -s /etc/nginx/sites-available/myproject /etc/nginx/sites-enabled/ ``` Then, reload Nginx to apply the changes: ```bash sudo systemctl reload nginx ``` ### Installing SSL with Certbot To secure your site with HTTPS, use Certbot to obtain an SSL certificate. First, install Certbot if you haven't already: ```bash sudo apt-get update sudo apt-get install certbot python3-certbot-nginx ``` Then, run Certbot with the following command, replacing `myproject.is-pro.dev` with your subdomain: ```bash sudo certbot --nginx -d myproject.is-pro.dev ``` Follow the prompts to complete the installation. Certbot will automatically configure Nginx to use the obtained SSL certificate. ## Common Pitfalls - **Incorrect Nginx Configuration**: Ensure your server block is correctly configured, especially the `server_name` and `proxy_pass` directives. - **Firewall Rules**: Make sure your VPS's firewall allows incoming traffic on ports 80 (HTTP) and 443 (HTTPS). - **Certbot Errors**: If Certbot fails to obtain a certificate, check your DNS settings and ensure your subdomain points to the VPS's IP address. ## Best Practices - **Regularly Update Your Server**: Keep your VPS and installed software up to date to protect against security vulnerabilities. - **Monitor Your Server**: Use tools like `htop` or `monit` to monitor your server's performance and respond to issues promptly. - **Backup Your Data**: Regularly backup your project files and database to prevent data loss in case of a failure. ## Conclusion Deploying your project to a VPS using SSH and a custom subdomain on is-pro.dev provides a flexible and scalable hosting solution. By following the steps outlined in this guide, you can securely host your application, leveraging Nginx, Certbot, and SSH. Remember to maintain your server, monitor its performance, and backup your data to ensure the longevity and reliability of your project. ## FAQ Q: How do I renew my SSL certificate? A: Certbot automatically renews SSL certificates. You can also manually renew by running `sudo certbot renew`. Q: What if my application runs on a different port? A: Update the `proxy_pass` directive in your Nginx configuration to point to the correct port, e.g., `http://localhost:8080`. Q: Can I host multiple projects on the same VPS? A: Yes, you can configure multiple server blocks in Nginx, each serving a different project on its own subdomain. Q: How do I troubleshoot Nginx configuration issues? A: Check the Nginx error log (usually located at `/var/log/nginx/error.log`) for clues, and test your configuration with `sudo nginx -t`.

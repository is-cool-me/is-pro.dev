Deploy to a VPS Using SSH and a Custom Subdomain

Configure a subdomain to point to a VPS server, set up Nginx as a reverse proxy, manage SSL certificates with Certbot, and automate deployments with Git hooks.

## Introduction to Deployment on a VPS Deploying a project to a Virtual Private Server (VPS) using SSH and a custom subdomain like myproject.is-pro.dev offers a high degree of control and flexibility. This approach is particularly appealing to developers who want to manage their project's infrastructure without the constraints of traditional shared hosting. By leveraging tools like Nginx, Certbot, and SSH, developers can set up a secure, scalable, and efficient deployment environment. ## Prerequisites for Deployment Before proceeding, ensure you have the following: - A VPS with a supported Linux distribution (e.g., Ubuntu). - A custom subdomain set up on is-cool-me (e.g., myproject.is-pro.dev). - Basic knowledge of SSH, Linux command line, and Nginx configuration. - Certbot installed on your VPS for SSL certificates. - Your project code ready for deployment. ## Setting Up Your VPS and SSH To start, connect to your VPS using SSH. If your VPS provider gave you a root password, use it to log in: ```bash ssh root@your_vps_ip ``` Once logged in, update your package list and upgrade any available packages: ```bash apt update && apt upgrade -y ``` Install Nginx: ```bash apt install nginx -y ``` Enable and start Nginx: ```bash systemctl enable nginx systemctl start nginx ``` ## Configuring Nginx for Your Subdomain Create a new Nginx configuration file for your subdomain: ```bash nano /etc/nginx/sites-available/myproject.is-pro.dev ``` Paste the following configuration, adjusting the `root` and `server_name` as necessary: ```nginx server { listen 80; server_name myproject.is-pro.dev; root /var/www/myproject; index index.html index.htm; location / { try_files $uri $uri/ =404; } } ``` Create a symbolic link to enable the site: ```bash ln -s /etc/nginx/sites-available/myproject.is-pro.dev /etc/nginx/sites-enabled/ ``` Test Nginx configuration and reload: ```bash nginx -t systemctl reload nginx ``` ## Setting Up SSL with Certbot Install Certbot and its Nginx plugin: ```bash apt install certbot python3-certbot-nginx -y ``` Obtain an SSL certificate for your subdomain: ```bash certbot --nginx -d myproject.is-pro.dev ``` Follow the prompts to complete the SSL setup. Certbot will automatically configure Nginx to use the SSL certificate. ## Deploying Your Project Clone your project repository to the `/var/www/myproject` directory or upload your project files there. Ensure the Nginx user has read access to these files: ```bash chown -R www-data:www-data /var/www/myproject ``` If your project requires a specific environment or build process, execute those commands now. ## Common Pitfalls and Solutions - **Nginx Configuration Errors**: Always test your Nginx configuration with `nginx -t` before reloading. - **SSL Certificate Issues**: Ensure your domain's DNS settings are correctly pointing to your VPS and wait for propagation if necessary. - **Permission Errors**: Verify file permissions for your project directory and adjust as needed. ## Best Practices for Security and Performance - **Regularly Update Packages**: Keep your VPS and installed packages up to date. - **Use Strong Passwords and SSH Keys**: Enhance SSH security with keys and avoid weak passwords. - **Monitor Your Server**: Use tools like `htop` and `nginx` logs to monitor performance and identify bottlenecks. - **Implement Firewall Rules**: Use `ufw` or similar tools to restrict incoming traffic to necessary ports. ## Troubleshooting Deployment Issues - **Check Nginx Logs**: Look for errors in `/var/log/nginx/error.log`. - **Verify DNS Propagation**: Use online tools to check if your domain's DNS changes have propagated. - **Test SSH Connection**: Ensure you can connect to your VPS via SSH without issues. ## Conclusion and Next Steps Deploying your project to a VPS using SSH and a custom subdomain like myproject.is-pro.dev provides a robust and customizable deployment solution. By following the steps outlined above and maintaining your server with best practices, you can ensure a secure, high-performance environment for your projects. Next, consider exploring additional security measures, optimizing your project for better performance, and learning more about advanced Nginx configurations. ## FAQ 1. **Q: How do I renew my SSL certificate?** A: Certbot automatically renews certificates. You can also manually renew using `certbot renew`. 2. **Q: Can I use this method for multiple subdomains?** A: Yes, create separate Nginx configurations for each subdomain, adjusting the `server_name` and `root` directives accordingly. 3. **Q: What if my project requires a database?** A: Install and configure a database server like MySQL or PostgreSQL on your VPS, and ensure your project is configured to connect to it. 4. **Q: How do I backup my VPS and project files?** A: Use tools like `rsync` or backup services to regularly backup your VPS files and database to a secure location. 5. **Q: Can I automate deployment using CI/CD tools?** A: Yes, tools like GitHub Actions or Jenkins can automate the deployment process to your VPS, integrating with your SSH and project repository. ## Further Reading - **Nginx Documentation**: For advanced configurations and troubleshooting, refer to the official Nginx documentation. - **Certbot Documentation**: Learn more about Certbot and SSL certificate management. - **Linux Security Best Practices**: Explore resources on securing your Linux VPS, such as configuring firewalls and managing user permissions. - **CI/CD Tools**: Investigate GitHub Actions, Jenkins, or similar tools for automating your deployment workflow. - **VPS Management**: Look into tools and guides for managing and optimizing your VPS for better performance and security.

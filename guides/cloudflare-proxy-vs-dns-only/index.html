Cloudflare Proxy vs DNS-Only: When to Use Each Mode

Understand the practical differences between Cloudflares proxy (orange cloud) and DNS-only (gray cloud) modes, and learn which to use for different types of subdomains.

Overview

Every DNS record on the is-pro.dev dashboard has a proxy toggle — the orange/gray cloud icon. This setting controls whether traffic flows through Cloudflare's edge network or directly to your origin server. This guide explains when to use each mode.

Orange Cloud — Proxied Mode

When the orange cloud is enabled, traffic reaches Cloudflare's edge first. Cloudflare terminates the SSL connection, caches static content, filters DDoS traffic, and forwards requests to your origin server. The origin sees Cloudflare's IP addresses, not the visitor's IP. Use this for HTTP/HTTPS web services, APIs, and any site needing security or performance enhancements.

Gray Cloud — DNS-Only Mode

When the gray cloud is enabled, DNS resolution returns your origin IP directly. Traffic bypasses Cloudflare entirely. Use this for non-HTTP services: email (MX), game servers (SRV), SSH, FTP, and any service that does not use HTTP or HTTPS on standard ports.

When Proxied Mode Causes Problems

Some services validate the client IP or require direct TCP connections. Shopify, for example, checks payment gateway callbacks from specific IPs. Netlify's Let's Encrypt verification can fail through Cloudflare proxy. For these services, use DNS-only mode.

Step 1: Toggle the Proxy Setting

On the is-pro.dev dashboard, find your subdomain's DNS record. Click the cloud icon to toggle between orange (proxied) and gray (DNS-only). The change propagates within seconds through Cloudflare's API. No TTL wait is needed.

Step 2: Verify the Change

Use dig yourname.is-pro.dev to check. In gray cloud (DNS-only) mode, dig returns your origin server IP. In orange cloud (proxied) mode, dig returns Cloudflare edge IPs (104.x.x.x, 172.x.x.x). For HTTP services, check the response headers — proxied responses include CF-Ray and Server: cloudflare headers.

Best Practices

  • Use proxied (orange) for all HTTP/HTTPS websites and APIs to get free SSL, DDoS protection, and caching
  • Use DNS-only (gray) for email, game servers, SSH, and non-HTTP services
  • During SSL certificate setup with third-party services, temporarily switch to gray cloud, then switch back once verified
  • Keep a record of which mode each subdomain uses — mix-ups cause hard-to-debug connectivity issues

FAQ

Does proxied mode hide my origin IP?

Yes — Cloudflare's edge IPs are visible to visitors, not your origin server IP. This prevents direct DDoS attacks on your server.

Can I proxy a subdomain that hosts email?

No — email traffic (SMTP, IMAP, POP3) does not use HTTP and cannot pass through Cloudflare proxy. Email subdomains must use DNS-only (gray cloud) mode.

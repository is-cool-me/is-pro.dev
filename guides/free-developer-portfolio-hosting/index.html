Build and Host a Developer Portfolio for Free

Create a professional developer portfolio using free tools, host it on GitHub Pages or Vercel, and connect it to a custom subdomain from is-pro.dev without spending a dime.

Overview

A portfolio on your own subdomain makes a strong impression on recruiters. This guide covers building a portfolio site with free tools and hosting it on an is-pro.dev subdomain with SSL.

Prerequisites

  • A GitHub account (for hosting or version control)
  • An is-pro.dev subdomain registered at dash.is-pro.dev
  • Portfolio content ready (projects, resume, contact info)

Step 1: Choose Your Portfolio Stack

For a simple one-page site, use plain HTML/CSS hosted on GitHub Pages. For more features, try Astro (component-based, zero JS by default) or Hugo (fast builds, great for project galleries). Each option deploys to free static hosting with a CNAME record pointing to your is-pro.dev subdomain.

Step 2: Build Your Portfolio

Include these sections: Hero (name, title, tagline), Projects (3–5 featured projects with descriptions and links), Skills (technology categories with proficiency), Experience (work history with bullet points), and Contact (email link or form). Use meta tags for Open Graph previews — recruiters often share portfolio links on social media.

Step 3: Configure DNS on is-pro.dev

Add a CNAME record on the is-pro.dev dashboard pointing your subdomain to your hosting target. For GitHub Pages: username.github.io. For Vercel: cname.vercel-dns.com. Enable Cloudflare proxy (orange cloud) for free SSL and DDoS protection.

Step 4: Add Analytics and SEO

Add a free analytics service like Plausible or Umami (self-hosted) to track portfolio visitors. Submit your subdomain to Google Search Console for indexing. Generate a sitemap.xml and include a robots.txt allowing all crawlers. Use structured data (JSON-LD) to add Portfolio schema for rich search results.

Best Practices

  • Keep the design minimal — let your projects speak for themselves
  • Include GitHub and LinkedIn links prominently in the header or hero section
  • Test mobile responsiveness — most recruiters browse portfolios on mobile
  • Add a PDF download link for your resume in the contact section

FAQ

Can I use a custom domain email with my portfolio?

Yes — see our guide on custom email setup. A hello@yourname.is-pro.dev address looks professional next to your portfolio URL.

Should I use a framework for my portfolio?

Only if you need interactivity. Plain HTML/CSS loads fastest and requires no build step. For animations or dynamic content, Astro is a lightweight choice that degrades gracefully.

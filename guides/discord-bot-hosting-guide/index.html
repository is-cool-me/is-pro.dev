Host a Discord Bot 24/7 on a Subdomain Endpoint

Expose your Discord bot dashboard or webhook endpoint through an is-pro.dev subdomain, configure SSL, and keep your bot running with uptime monitoring and proper DNS routing.

Overview

Discord bots need a reachable endpoint for webhooks, command APIs, or admin dashboards. By routing traffic through an is-pro.dev subdomain, you get a stable URL with free SSL for your bot's HTTP services.

Prerequisites

  • A Discord bot created in the Discord Developer Portal with a token
  • Hosting for the bot (Fly.io, Railway, or a VPS)
  • An is-pro.dev subdomain registered at dash.is-pro.dev

Step 1: Deploy Your Bot to a Cloud Service

Use Fly.io for free-tier hosting: create a fly.toml with internal_port = 8080 and deploy with fly deploy. Fly.io assigns a *.fly.dev URL. Alternatively, use Railway with a Git-connected deployment — it provides a *.railway.app URL automatically.

Step 2: Configure DNS on is-pro.dev

Add a CNAME record on the is-pro.dev dashboard pointing your subdomain to your hosting target (e.g. your-bot.fly.dev or your-bot.up.railway.app). Enable Cloudflare proxy (orange cloud) for SSL termination and DDoS protection.

Step 3: Set Up Webhook Endpoints

Configure your bot code to listen for incoming webhooks on the hosted port. Use Express.js or similar to create routes like POST /webhook/discord. Ensure your bot validates incoming requests using signature verification to prevent spoofing.

Step 4: Configure Uptime Monitoring

Set up a free monitoring service like UptimeRobot or Better Stack to ping your subdomain every 5 minutes. Configure alerts via Discord webhook (your own bot channel) or email to catch downtime early. Most free tiers allow 50 monitors.

Best Practices

  • Store the Discord bot token in environment variables, never in code
  • Use rate limiting middleware (e.g., express-rate-limit) on webhook endpoints
  • Enable Cloudflare's Bot Fight Mode to block automated abuse
  • Keep the bot process alive with a process manager like pm2 or systemd

FAQ

Do I need a static IP for my bot?

No — CNAME records resolve through DNS, so your backend IP can change without affecting the subdomain. Cloudflare proxy hides your origin IP.

Can I host multiple bots on one subdomain?

Yes — use path-based routing (/bot1/webhook, /bot2/webhook) or run separate processes behind a reverse proxy like Nginx or Caddy.

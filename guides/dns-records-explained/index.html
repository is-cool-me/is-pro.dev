DNS Record Reference

DNS records map domain names to IP addresses, mail servers, and other services. Here is every record type available on the is-pro.dev dashboard and when to use it.

A and AAAA Records

Point a subdomain directly to an IPv4 (A) or IPv6 (AAAA) address. Use these when hosting on a VPS or dedicated server with a static IP. Example: @ 3600 IN A 203.0.113.10. For IPv6: @ 3600 IN AAAA 2001:db8::1.

CNAME Records

Alias one domain to another. Use CNAME to point your subdomain to a platform's target hostname like username.github.io or cname.vercel-dns.com. CNAME records cannot coexist with other record types at the same name.

MX Records

Direct email for your subdomain to a mail server. Each MX record has a priority value — lower numbers are tried first. Example: @ 3600 IN MX 10 mail.yourname.is-pro.dev. Always pair MX with SPF and DKIM to avoid spam filtering.

TXT, SPF, DKIM, and DMARC

TXT records hold arbitrary text data. SPF (TXT type) lists servers authorized to send email for your domain: v=spf1 include:_spf.google.com ~all. DKIM stores a public key for email signing verification. DMARC tells receivers how to handle unauthenticated email (v=DMARC1; p=quarantine;).

SRV Records

Specify a host and port for specific services. Used by Minecraft (_minecraft._tcp 3600 IN SRV 0 5 25565 mc.example.com), Discord, and SIP. The format is: priority, weight, port, target hostname.

CAA Records

Restrict which Certificate Authorities can issue SSL certificates for your domain. Example: @ 3600 IN CAA 0 issue "letsencrypt.org". Prevents unauthorized certificate issuance.

Best Practices

  • Use CNAME instead of A records when the target IP may change
  • Set TTL low (60–300 seconds) during migrations, high (3600+) for stable configs
  • Always publish SPF, DKIM, and DMARC if you send email from the subdomain
  • Use SRV records for any service running on a non-standard port

FAQ

Can I have a CNAME and MX on the same subdomain?

No — CNAME conflicts with other record types. Use an A record instead if you need both web and email on the same subdomain.

How fast do DNS changes propagate?

Cloudflare propagates changes within seconds. End users see updates once their resolver's TTL cache expires.

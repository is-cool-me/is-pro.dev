Host Next.js on Cloudflare Pages with Free Subdomain

Complete guide to deploying Next.js applications on Cloudflare Pages, configuring DNS records through is-pro.dev, and enabling edge caching for fast global performance.

Overview

Cloudflare Pages hosts static and server-side rendered sites on Cloudflare's global edge network. With Next.js support via @cloudflare/next-on-pages, you get sub-second page loads worldwide.

Prerequisites

  • A Next.js project (version 13+ recommended)
  • A GitHub or GitLab account
  • An is-pro.dev subdomain registered at dash.is-pro.dev
  • Node.js 18+ installed locally

Step 1: Configure Next.js for Cloudflare Pages

Install the Cloudflare Pages adapter: npm install @cloudflare/next-on-pages. Update next.config.js to use output: "standalone" and add experimental: { runtime: "edge" } for server-side features. Run npx @cloudflare/next-on-pages to verify the build works locally.

Step 2: Deploy to Cloudflare Pages

In the Cloudflare dashboard, create a new Pages project connected to your Git repository. Set the build command to npx @cloudflare/next-on-pages and the output directory to .vercel/output/static. After the first deploy, Cloudflare assigns a *.pages.dev URL for testing.

Step 3: Configure DNS on is-pro.dev

Add a CNAME record on the is-pro.dev dashboard pointing your subdomain to your Cloudflare Pages project URL (e.g. my-next-app.pages.dev). Keep the Cloudflare proxy (orange cloud) enabled for DDoS protection and SSL.

Step 4: Add Custom Domain in Cloudflare Pages

In Pages → your project → Custom Domains, enter your is-pro.dev subdomain. Cloudflare verifies ownership through DNS. Once verified, a free SSL certificate is provisioned automatically. Enable "Always Use HTTPS" in the SSL/TLS settings.

Best Practices

  • Use getStaticProps for content that does not change often to maximize edge caching
  • For dynamic routes, configure incrementalStaticRegeneration: true in next.config.js
  • Monitor edge function execution time — keep responses under 10ms for best performance
  • Use Cloudflare's Cache Reserve for large static assets like images and fonts

FAQ

Does Cloudflare Pages support Next.js App Router?

Yes. The @cloudflare/next-on-pages adapter supports both Pages Router and App Router, including server components and route handlers.

Can I use serverless functions with Next.js on Cloudflare Pages?

Yes — API routes and server actions run as Cloudflare Workers with edge runtime. Note that Node.js-specific APIs like fs are not available at the edge.

Redirect an Old Domain to Your New Subdomain Permanently

Set up 301 permanent redirects to send visitors from an old domain to your new is-pro.dev subdomain while preserving search engine rankings and maintaining analytics.

Overview

301 redirects tell search engines and browsers that a page has permanently moved. This guide covers redirecting from an old is-pro.dev subdomain (or external domain) to a new one while preserving SEO value.

Prerequisites

  • Access to the DNS settings for both the old and new domains
  • An is-pro.dev subdomain registered as the new target
  • A hosting provider that supports redirect configuration (or Cloudflare)

Step 1: Configure the Old Domain DNS

If the old domain is also an is-pro.dev subdomain: update its DNS record to point to a server that can issue redirects. The simplest approach is to use Cloudflare's Bulk Redirect feature (available with proxied DNS). On the is-pro.dev dashboard, enable Cloudflare proxy (orange cloud) for the old subdomain's record.

Step 2: Set Up Cloudflare Bulk Redirects

In the Cloudflare dashboard, go to Rules → Bulk Redirects. Create a new redirect rule: source URL https://old.yourname.is-pro.dev/* → target URL https://new.yourname.is-pro.dev/$1 with status code 301 (Permanent Redirect). The $1 captures the path, preserving the full URL structure. Toggle "Preserve query string" ON.

Step 3: Alternative — Redirect with Page Rules

If you have fewer than 3 redirects, use Cloudflare Page Rules instead. Create a rule: URL matches old.yourname.is-pro.dev/*, setting "Forwarding URL" with status 301 to https://new.yourname.is-pro.dev/$1. Free Cloudflare plans include 3 Page Rules.

Step 4: Update Internal Links

After the redirect is live, update all internal links, API clients, and documentation to point directly to the new subdomain. Relying on redirects long-term creates latency and complicates debugging. Use link-checking tools like broken-link-checker (npm) to find remaining old links.

Best Practices

  • Use 301 (permanent) redirects, not 302 (temporary) — 301 passes SEO ranking to the new URL
  • Keep the old subdomain active for at least 6 months after the migration
  • Update your sitemap.xml to list only the new subdomain's URLs
  • Notify Google via Google Search Console's URL Change tool for faster reindexing

FAQ

Will I lose search rankings after a 301 redirect?

Google passes most ranking signals through a 301 redirect — on average, sites retain 90–99% of their ranking after a properly configured permanent redirect.

Can I redirect to a subpath of the new domain?

Yes — configure the redirect target as https://new.yourname.is-pro.dev/specific-page. Use * wildcards for pattern-based redirects.

Netlify Custom Domain Setup with Free Subdomain

Connect your is-pro.dev subdomain to a Netlify site, configure DNS records for both root and www variants, and set up automatic HTTPS provisioning through Netlify Lets Encrypt.

Overview

Netlify offers Git-based deployments with built-in form handling, serverless functions, and automatic HTTPS. This guide walks through connecting an is-pro.dev subdomain to your Netlify site.

Prerequisites

  • A Netlify account and a deployed site
  • A registered is-pro.dev subdomain
  • Access to the is-pro.dev dashboard for DNS management

Step 1: Get Your Netlify Site Domain

In the Netlify dashboard, go to Site settings → Domain management. Netlify assigns a default URL like random-name-123456.netlify.app. Copy this URL — you will use it as the CNAME target for your is-pro.dev subdomain.

Step 2: Configure DNS on is-pro.dev

Add a CNAME record on the is-pro.dev dashboard pointing your subdomain to your Netlify site URL (e.g. my-site.netlify.app). Do not use Cloudflare proxy (gray cloud) — Netlify manages its own SSL through Let's Encrypt and proxying through Cloudflare can cause certificate validation failures.

Step 3: Add Custom Domain in Netlify

In Netlify's Domain management, click "Add custom domain". Enter your full is-pro.dev subdomain. Netlify checks the DNS record — once it detects the CNAME, it provisions a Let's Encrypt certificate automatically. Certificate issuance takes 1–15 minutes.

Step 4: Enable HTTPS and Set Primary Domain

Once the certificate is active, Netlify shows "Netlify Certificate" as the SSL provider. Set your is-pro.dev subdomain as the primary domain. Enable "Force HTTPS" in the HTTPS settings to redirect all HTTP traffic to HTTPS with a 301 redirect.

Best Practices

  • Keep Cloudflare DNS-only (gray cloud) — do not proxy through Cloudflare
  • Set up branch-based deploys for staging: staging.yourname.is-pro.dev → staging branch
  • Use Netlify's split testing for gradual rollouts of layout changes
  • Enable Deploy Previews for PR-based testing before merging to production

FAQ

Can I use Cloudflare proxy with Netlify?

Technically yes, but it is not recommended — Cloudflare can interfere with Let's Encrypt verification. If you must proxy, set SSL/TLS to "Full (Strict)" and pause Cloudflare proxy during initial certificate issuance.

Does Netlify support wildcard subdomains?

No — Netlify requires explicit domain entries for each subdomain. Add each subdomain individually in the Netlify dashboard.

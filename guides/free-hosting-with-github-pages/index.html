Free Hosting with GitHub Pages: Complete Setup Guide

Learn how to host your static website for free using GitHub Pages, configure custom domains with is-pro.dev, and set up automatic deployments from your repository.

Overview

GitHub Pages serves static sites directly from a GitHub repository with built-in HTTPS and custom domain support. Combined with an is-pro.dev subdomain, you get a production-ready URL at no cost.

Prerequisites

  • A GitHub account with a repository for your site
  • An is-pro.dev subdomain registered at dash.is-pro.dev
  • Your site files (HTML, CSS, JS) ready to push

Step 1: Create Your GitHub Pages Site

Push your static files to a GitHub repository. Go to Settings → Pages and set the source branch — typically main with the / (root) folder. GitHub assigns a default URL like username.github.io/repo-name. Verify this works before adding your custom subdomain.

Step 2: Configure DNS on is-pro.dev

On the is-pro.dev dashboard, create a CNAME record pointing your subdomain to username.github.io. Do not use an A record — GitHub Pages only accepts CNAME-based custom domains for project sites. Make sure the Cloudflare proxy (orange cloud) is enabled so SSL is handled automatically.

Step 3: Add the CNAME File

In your repository root, create a file named CNAME (no extension) containing your full subdomain, e.g. myproject.yourname.is-pro.dev. Commit and push this file. Alternatively, enter the custom domain in the GitHub Pages settings and let GitHub create the file for you.

Step 4: Enable HTTPS in GitHub Settings

After a few minutes, GitHub will attempt to provision an SSL certificate. In Settings → Pages, check the "Enforce HTTPS" checkbox once it becomes available. Certificate issuance can take up to 15 minutes — DNS must resolve correctly first.

Best Practices

  • Commit the CNAME file to your repo so the domain persists across rebuilds
  • Use a .nojekyll file if your site uses underscores in folder names
  • Clear GitHub's build cache if your site does not update after a push
  • Keep your is-pro.dev subdomain registration active to avoid domain loss

Troubleshooting

SSL pending forever: Ensure the CNAME record points to username.github.io (not github.com) and Cloudflare proxy is enabled. Disable and re-enable "Enforce HTTPS" after 10 minutes.

404 after deploy: Check your repository's Pages source path — if using a /docs folder or a gh-pages branch, make sure the path matches.

FAQ

Can I use an apex domain with GitHub Pages?

GitHub Pages supports apex domains through A records pointing to GitHub's IP addresses, but is-pro.dev subdomains use CNAME which is simpler and recommended.

Does GitHub Pages support custom 404 pages?

Yes. Create a 404.html in your repository root and GitHub serves it automatically for any missing page.

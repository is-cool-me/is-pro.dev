SSL Certificate Setup: From Zero to HTTPS Everywhere

Learn how SSL certificates work, configure free auto-renewing certificates through Cloudflare, and ensure every page on your subdomain forces HTTPS with proper redirect rules.

Overview

All is-pro.dev subdomains include free SSL through Cloudflare. This guide explains how SSL provisioning works, how to troubleshoot common issues, and how to configure Full (Strict) encryption for maximum security.

How SSL Works on is-pro.dev

When Cloudflare proxy (orange cloud) is enabled, Cloudflare issues a free, auto-renewing SSL certificate for your subdomain. The certificate covers both the root and www variants. Cloudflare terminates the SSL connection at the edge and re-encrypts traffic to your origin server using a separate origin certificate.

Step 1: Enable Cloudflare Proxy

On the is-pro.dev dashboard, ensure your subdomain's DNS record has the orange cloud icon (proxied). This activates Cloudflare's edge SSL. If you see a gray cloud, click it to toggle to orange. SSL certificate issuance begins automatically once the record is proxied.

Step 2: Configure SSL/TLS Mode

In the Cloudflare dashboard, go to SSL/TLS → Overview. For most setups, "Full (Strict)" is recommended — Cloudflare encrypts traffic to your origin server and verifies the origin certificate. "Flexible" mode (encrypts browser-to-Cloudflare only) is less secure and should only be used during initial setup.

Step 3: Force HTTPS with Page Rules

Create a Cloudflare Page Rule: URL matches *yourname.is-pro.dev/*, setting "Always Use HTTPS" to ON. This redirects all HTTP requests to HTTPS with a 301 redirect. For HSTS, add a security header: Strict-Transport-Security: max-age=31536000; includeSubDomains; preload.

Step 4: Troubleshoot Common Issues

SSL pending: If the certificate shows "Pending" for over an hour, ensure your DNS record is proxied (orange cloud). Disable and re-enable proxy to trigger re-issuance.

Certificate mismatch: Make sure your subdomain's CNAME or A record resolves to the correct target. A mismatch causes Cloudflare to show a "This site can't provide a secure connection" error.

Mixed content warnings: Use relative URLs (/images/logo.png) or protocol-relative URLs (//cdn.example.com/file.js) to avoid HTTPS/HTTP content mix.

FAQ

Does Cloudflare SSL work with non-standard ports?

Cloudflare proxy only supports standard HTTP/HTTPS ports (80, 443). For other ports, use DNS-only mode (gray cloud) and handle SSL on your origin server.

Can I use my own certificate instead of Cloudflare's?

Yes — Cloudflare supports custom certificates via the SSL/TLS → Edge Certificates page. Upload your certificate and private key, then enable "Full (Strict)" mode.

Host WordPress on a Free is-pro.dev Subdomain

Deploy a WordPress site using PHP hosting services like 000webhost or by connecting to a VPS, configure your is-pro.dev subdomain for WordPress-specific DNS requirements.

Overview

WordPress requires PHP and MySQL — it cannot run on static hosting. This guide covers two approaches: using a free PHP hosting service or setting up WordPress on a VPS with your is-pro.dev subdomain.

Prerequisites

  • A PHP hosting account or a VPS with root access
  • WordPress downloaded from wordpress.org
  • An is-pro.dev subdomain registered at dash.is-pro.dev

Option A: Free PHP Hosting

Services like 000webhost or InfinityFree offer free PHP 8 support with MySQL. Upload WordPress via their file manager or FTP client. Create a MySQL database through the hosting panel and configure wp-config.php with the database credentials. Most free hosts assign a temporary URL for initial setup.

Option B: VPS Deployment

On a VPS, install Nginx, PHP 8.x, and MariaDB: apt install nginx php-fpm php-mysql mariadb-server. Create a database with mysql -e "CREATE DATABASE wordpress". Download WordPress: wget wordpress.org/latest.tar.gz && tar xzf latest.tar.gz. Configure the Nginx site block to point to the WordPress directory.

Step 3: Configure DNS on is-pro.dev

Add an A record on the is-pro.dev dashboard pointing your subdomain to your server's public IP. If using 000webhost, they provide a target IP or CNAME. Enable Cloudflare proxy for free SSL — WordPress admin panels benefit from DDoS protection.

Step 4: Configure WordPress Site URL

In the WordPress admin dashboard (Settings → General), set "WordPress Address" and "Site Address" to your full is-pro.dev subdomain URL (e.g. https://blog.yourname.is-pro.dev). If you migrated from a temporary URL, update the wp-config.php with define('WP_HOME','https://...') and define('WP_SITEURL','https://...').

Best Practices

  • Install a security plugin like Wordfence or Sucuri for firewall protection
  • Use strong passwords and disable the default "admin" username
  • Enable automatic updates for WordPress core and plugins
  • Set up a Redis or file-based caching plugin to reduce server load

FAQ

Can I use Cloudflare's APO for WordPress?

Yes — Cloudflare's Automatic Platform Optimization (APO) caches HTML pages at the edge. It requires a paid plan but significantly improves TTFB for global visitors.

Does WordPress work on static hosting?

No — WordPress requires PHP and a database. Use a PHP host or VPS. For a static alternative, consider exporting WordPress content via the "Simply Static" plugin to host on GitHub Pages.

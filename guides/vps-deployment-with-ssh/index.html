Deploy to a VPS Using SSH and a Custom Subdomain

Configure a subdomain to point to a VPS server, set up Nginx as a reverse proxy, manage SSL certificates with Certbot, and automate deployments with Git hooks.

Overview

A VPS gives you full control over your stack. This guide covers pointing your is-pro.dev subdomain to a VPS, setting up Nginx, securing traffic with Let's Encrypt, and automating deployments via Git hooks.

Prerequisites

  • A VPS with Ubuntu/Debian and SSH access
  • A registered is-pro.dev subdomain
  • Basic Linux command-line knowledge

Step 1: Configure DNS on is-pro.dev

Add an A record on the is-pro.dev dashboard pointing your subdomain to your VPS's public IPv4 address. Enable Cloudflare proxy (orange cloud) for DDoS protection and SSL termination. If your VPS uses IPv6, also add an AAAA record.

Step 2: Install Nginx and Certbot

SSH into your VPS: ssh root@your-vps-ip. Install Nginx: apt update && apt install nginx certbot python3-certbot-nginx. Start Nginx: systemctl enable --now nginx. Create a site config in /etc/nginx/sites-available/your-subdomain with a server_name matching your is-pro.dev subdomain.

Step 3: Obtain an SSL Certificate

Run Certbot: certbot --nginx -d yourname.is-pro.dev. Certbot automatically modifies your Nginx config to redirect HTTP to HTTPS and provisions a Let's Encrypt certificate. Certificates auto-renew via a systemd timer. Verify: certbot renew --dry-run.

Step 4: Set Up Automated Deployments

Create a bare Git repository on the VPS: mkdir -p /var/repo/site.git && cd /var/repo/site.git && git init --bare. Add a post-receive hook that checks out files to /var/www/ and reloads Nginx. On your local machine, add the VPS as a remote: git remote add production ssh://root@your-vps-ip/var/repo/site.git. Push to deploy: git push production main.

Best Practices

  • Use SSH key authentication only — disable password login in /etc/ssh/sshd_config
  • Set up UFW: ufw allow 22/tcp && ufw allow 80/tcp && ufw allow 443/tcp && ufw enable
  • Monitor resources with htop and df -h — set up disk usage alerts
  • Keep the system updated with unattended-upgrades for security patches

FAQ

Can I host multiple subdomains on one VPS?

Yes — add multiple server blocks in Nginx, one per subdomain. Use server_name directives and separate root directories for each site.

What if my VPS IP changes?

Update the A record in the is-pro.dev dashboard with the new IP. DNS propagation typically takes under a minute with Cloudflare.

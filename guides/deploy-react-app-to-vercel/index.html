Deploy a React App to Vercel with Custom Domain

Step-by-step guide to deploying your React application to Vercel, connecting a free is-pro.dev subdomain, and configuring environment variables for production.

Overview

Vercel provides instant deployments from Git, serverless functions, and automatic SSL for custom domains. This guide walks through deploying a React app and connecting it to an is-pro.dev subdomain.

Prerequisites

  • A React project ready for deployment (created with Vite, CRA, or Next.js)
  • A GitHub, GitLab, or Bitbucket account
  • An is-pro.dev subdomain registered at dash.is-pro.dev

Step 1: Import Your Repository to Vercel

Log in to vercel.com and click "Add New → Project". Select your Git provider and repository. Vercel auto-detects the framework (React, Vite, Next.js) and sets the build command and output directory. Click "Deploy" to run the first build.

Step 2: Add a Custom Domain in Vercel

After deployment, go to your project's Settings → Domains. Enter your is-pro.dev subdomain (e.g. myapp.yourname.is-pro.dev). Vercel shows the required DNS configuration — typically a CNAME record pointing to cname.vercel-dns.com.

Step 3: Configure DNS on is-pro.dev

On the is-pro.dev dashboard, add a CNAME record for your subdomain targeting cname.vercel-dns.com. Keep the Cloudflare proxy (orange cloud) enabled to get free SSL. Vercel will detect the DNS change and automatically provision a certificate.

Step 4: Verify and Set Environment Variables

Visit your subdomain — the React app should load with a valid SSL certificate. In Vercel's project settings, add any required environment variables (API_URL, DATABASE_URL, etc.) and redeploy. Use Vercel's Preview Deployments for testing before promoting to production.

Best Practices

  • Set framework: "vite" or framework: "create-react-app" in vercel.json for optimal build settings
  • Use Vercel's environment variable groups to manage secrets across projects
  • Enable "Automatic HTTPS Redirects" in Vercel DNS settings
  • Set up a Production and Preview branch in Vercel for safe deployments

FAQ

Does Vercel support SPA client-side routing?

Yes. Add a vercel.json with {"rewrites": [{"source": "/(.*)", "destination": "/index.html"}]} to handle React Router or Vue Router routes.

How do I set up a monorepo with multiple apps?

Vercel supports monorepos — configure each app in the Vercel dashboard with its own root directory and build command.

Set Up Uptime Monitoring for Your Subdomain Services

Monitor your subdomains for downtime using free tools like UptimeRobot or Better Stack, configure alerting webhooks, and use the data to maintain a reliable service.

Overview

Uptime monitoring ensures your subdomain—pointing to your site, API, or server—stays accessible. This guide covers setting up free monitors with alerting for your is-pro.dev subdomains.

Prerequisites

  • An is-pro.dev subdomain with an active site or service
  • A free account on UptimeRobot or Better Stack
  • (Optional) A Discord webhook URL for alerts

Step 1: Create a Monitor

In UptimeRobot, click "Add New Monitor". Select "HTTP(s)" as the monitor type. Enter your is-pro.dev subdomain URL (e.g. https://myapp.yourname.is-pro.dev). Set the check interval to 5 minutes (free tier). UptimeRobot pings your URL from multiple global locations every 5 minutes.

Step 2: Configure Alerting

In the same monitor settings, add alert contacts. UptimeRobot sends alerts via email by default. For real-time alerts, add a Slack or Discord webhook: get your Discord channel's webhook URL from Channel Settings → Integrations → Webhooks, then paste it in UptimeRobot's alert contact settings.

Step 3: Set Up Status Page

Better Stack offers a free public status page. Create one and link your monitors to it. Share the status page URL — https://betterstack.com/status/your-page — with your users. The status page shows live uptime percentages, response times, and incident history for each monitored subdomain.

Step 4: Monitor SSL Expiry

Add an SSL certificate monitor in UptimeRobot or use a dedicated SSL checker. The monitor warns you 30, 14, and 7 days before your Cloudflare certificate expires. Note that Cloudflare certificates auto-renew, but the monitor catches cases where DNS misconfiguration prevents renewal.

Best Practices

  • Monitor from multiple geographic regions to detect regional outages
  • Set alert thresholds — only notify after 2 consecutive failures to avoid false alarms
  • Log response times to track performance regressions after deployments
  • Create a runbook for common failure scenarios (DNS misconfig, origin down, SSL expired)

FAQ

Does monitoring affect my credit usage?

No — monitoring services ping your subdomain as normal HTTP requests. There is no charge for incoming traffic on is-pro.dev subdomains.

Can I monitor internal services that are not public?

No — monitoring services need public HTTP access. For internal services, use a self-hosted monitoring solution like Prometheus with Blackbox Exporter.
